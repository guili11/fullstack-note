Go 数据库核心依赖：操作库 + 驱动
操作库（database/sql）
身份：Go 官方标准库，通用接口层
作用：定义统一的数据库操作规则（连接、CURD、事务、连接池）
核心：屏蔽数据库差异，提供通用能力
数据库驱动（如 mysql 驱动）
身份：底层实现层
作用：适配具体数据库，实现协议通信、格式转换
核心：让通用接口能真正连接并操作数据库
二者是核心的原因
解耦架构：一套代码适配所有数据库
职责分离：通用逻辑与底层实现解耦，易维护
生态根基：所有 ORM（GORM）均依赖此架构

# 二、每一层到底写什么（重点）

## 0. 认知模型（非常关键）

> 这一层决定你是否只是“会用”，还是“理解数据库”

* 数据库是什么（存储 + 查询引擎）
* MySQL 架构分层（Server 层 / 存储引擎层）
* 一条 SQL 执行流程（解析 → 优化 → 执行）
* 数据 vs 索引 vs 页（page）的关系
* OLTP vs OLAP（简单认知即可）

👉 这一层建议你写“自己的理解”，不是抄文档

---

## 1. SQL基础（但要工程化整理）

不要写成语法手册，要写成“使用模式”

```
CRUD
JOIN（重点：各种 join 使用场景）
聚合（group by + having）
子查询 vs join
排序 + 分页（limit offset 问题）
```

👉 每一类：

* 写语法 ✔
* 写“什么时候用” ✔
* 写“坑” ✔

---

## 2. 数据建模（你未来最值钱的能力之一）

```
范式（1NF / 2NF / 3NF）
反范式（什么时候要冗余）
一对一 / 一对多 / 多对多设计
字段设计（类型选择、是否允许 null）
表设计规范（命名、主键策略）
```

👉 强烈建议：

* 每个模型配一个实际例子（用户系统、电商等）

---

## 3. MySQL核心机制（开始拉开差距）

```
存储引擎（InnoDB 为主）
数据页结构
B+Tree（核心）
redo log / undo log
binlog（了解即可）
```

👉 这里不要追求全懂，但要有：

* “概念 + 作用 + 解决什么问题”

---

## 4. 查询优化（面试 + 实战核心）

```
EXPLAIN 使用
索引是否命中
覆盖索引
回表
最左前缀原则
慢查询分析
```

👉 每一个点都写：

* 示例 SQL
* 执行结果
* 为什么快/慢

---

## 5. 事务与并发控制

```
事务ACID

脏读 / 不可重复读 / 幻读

锁（行锁 / 表锁 / 间隙锁）
```

👉 这里建议画图（很重要）

## 6. 索引体系（必须单独一层）
