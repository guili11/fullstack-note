# 🚦 前端核心问题清单（工程级视角）

1. **状态管理**

   * 全局 vs 局部状态
   * store 层（Pinia / Zustand / Redux 等）
   * 响应式 vs 非响应式
   * 状态更新策略（immutable / mutable）

2. **逻辑分层**

   * view / controller / store / service
   * 函数放置规则
   * UI 逻辑 vs 控制逻辑 vs 数据逻辑
   * 可复用性 & 组合性

3. **错误处理**

   * service 抛出
   * controller 捕获
   * view 显示
   * utils 统一处理
   * 提示/回滚/乐观更新策略

4. **副作用管理（你下一步）**

   * 数据请求 / fetch / axios
   * 表单填充 / UI 初始化
   * 订阅 / SSE / WebSocket
   * timer / setInterval / animation
   * 清理副作用（unmount / reset / cancel）
   * useEffect / watch / onMounted 等控制依赖
   * 副作用依赖表（哪些依赖谁变化才触发）

5. **表单管理**

   * form state 与 store 的关系
   * 新建 vs 编辑模式初始化
   * 校验逻辑（同步 / 异步）
   * 重置与提交策略
   * 复杂表单拆分（可复用组件）

6. **事件与交互**

   * 事件冒泡与阻止
   * 可维护的事件绑定（子组件 vs 父组件）
   * debounce / throttle / 防抖处理
   * 按需事件监听（避免全局监听）

7. **异步请求 & 网络策略**

   * loading 状态管理
   * 并发 / 排队 / 取消请求
   * 重试 & 超时策略
   * 乐观更新 & 回滚

8. **性能 & 渲染优化**

   * 组件拆分 / memo / computed / reactive
   * 大列表 / 虚拟列表
   * 不必要的副作用触发
   * 渲染防抖 / 节流

9. **复用 & 可维护性**

   * 通用组件 / hook / util
   * 统一接口与契约
   * 代码分层 & 模块化
   * 命名规范与约定

10. **安全 & 稳定性**

    * 数据校验 / 类型约束
    * 防止内存泄漏（副作用清理）
    * 异步错误捕获
    * 全局错误边界 / fallback
