# ✅ 二、四层职责（精确版）

***

## 1️⃣ view（视图层）

```txt
只负责：UI + 事件触发
```

### 放什么：

```txt
✔ JSX / template
✔ 表单 UI
✔ 点击事件（onClick）
✔ 调用 controller
```

***

### ❌ 不放：

```txt
❌ 请求
❌ 业务逻辑
❌ 状态管理
```

***

## 2️⃣ controller（hooks  控制层 ⭐核心）

```txt
负责：业务流程编排+管理局部业务状态
```

***

### 放什么函数：

```txt
✔ submit(values)
✔ fetchList()
✔ openModal / closeModal
✔ 调用useState（）  管理局部业务状态
✔ 调用 service
✔ 操作 store
✔ 错误处理（核心位置）
```

***

### 本质：

```txt
controller = “大脑”
```

***

## 3️⃣ store（全局状态层）

```txt
负责：存数据，不做流程
```

***

### 放什么：

```txt
✔ list
✔ loading
✔ currentCourse
✔ open
```

***

### 写法：

```ts
export const useCourseStore = create((set) => ({
  list: [],
  loading: false,
  open: false,
  currentCourse: null,

  setList: (list) => set({ list }),
  setLoading: (loading) => set({ loading }),
  setOpen: (open) => set({ open }),
  setCurrentCourse: (course) => set({ currentCourse: course }),
}))
```

***

### ❗原则：

```txt
store = 数据仓库（不能有业务逻辑）
```

***

## 4️⃣ service（数据层）

```txt
负责：请求 API
```

***

### 放什么：

```ts
getCourseList()
createCourse()
updateCourse()
deleteCourse()
```

***

### ❗原则：

```txt
✔ 只请求
✔ 不关心 UI
✔ 不处理业务流程
```

***

# ✅ 三、核心流转（你必须理解）

```txt
view → controller → service → controller → store → view
```

👉 翻译成一句话：

```txt
UI 触发 → controller 决策 → 调接口 → 更新 store → UI 自动更新
```

# ✅ 四、关键代码示例（完整一条链）

***

## 1️⃣ service

```ts
export async function createCourse(data) {
  const res = await request.post('/course', data)

  if (res.code !== 0) {
    throw new Error(res.message)
  }

  return res.data
}
```

***

## 2️⃣ store

```ts
const useCourseStore = create((set) => ({
  list: [],
  loading: false,

  setList: (list) => set({ list }),
  setLoading: (loading) => set({ loading }),
}))
```

***

## 3️⃣ controller（核心）

```ts
export function useCourseController() {
  const {
    list,
    loading,
    setList,
    setLoading,
  } = useCourseStore()

  async function fetchList() {
    setLoading(true)
    try {
      const data = await getCourseList()
      setList(data)
    } catch (e) {
      handleError(e)
    } finally {
      setLoading(false)
    }
  }

  async function submit(values) {
    setLoading(true)
    try {
      await createCourse(values)
      await fetchList()
    } catch (e) {
      handleError(e)
    } finally {
      setLoading(false)
    }
  }

  return {
    list,
    loading,
    fetchList,
    submit,
  }
}
```

***

## 4️⃣ view

```tsx
function CoursePage() {
  const {
    list,
    loading,
    submit,
  } = useCourseController()

  function handleSubmit() {
    submit({ name: 'xxx' })
  }

  return (
    <>
      <button onClick={handleSubmit}>提交</button>
      {loading && <span>loading...</span>}
    </>
  )
}
```

# ✅ 五、错误处理（四层版）

***

## ❗规则：

```txt
service：抛错误
controller：处理错误（核心）
view：只展示
store：不碰错误
```

***

## 实现：

### service

```ts
throw new Error('创建失败')
```

***

### controller

```ts
catch (e) {
  handleError(e)
}
```

***

### utils/error.ts

```ts
export function handleError(error) {
  console.error(error)
  message.error(error.message || '请求失败')
}
```
