# 请求层封装（Request Layer）

## 一、这一层解决什么问题？

前端所有请求相关逻辑的统一入口，包括：

- 请求发送（fetch / axios）
- token 注入
- 错误统一处理
- 跨域处理（开发环境）
- 请求参数规范

---

## 二、为什么要封装？

如果不封装，会出现：

- 每个组件自己写请求 ❌
- token 到处写 ❌
- 错误处理不统一 ❌

👉 导致：

- 代码重复
- 难以维护
- 不可扩展

---

## 三、职责划分（核心）

```text
组件（UI）
   ↓
controller（业务流程）
   ↓
service（调用请求）
   ↓
请求层封装 ⭐（统一处理）
   ↓
后端 API
```
