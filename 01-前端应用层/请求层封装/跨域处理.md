# 跨域处理（前端方案）

## 一、问题背景

浏览器存在**同源策略（Same-Origin Policy）**：

```
协议 + 域名 + 端口 必须一致
```

否则请求会被浏览器拦截（跨域）

---

## 二、解决方案（开发环境）

使用 **Vite proxy** 解决跨域问题

---

## 三、配置方法

### 1️⃣ vite.config.ts

```ts
import { defineConfig } from 'vite'

export default defineConfig({
  server: {
    proxy: {
      '/api': {
        target: 'http://localhost:3000',
        changeOrigin: true,
        secure: false,
        rewrite: (path) => path.replace(/^\/api/, '')
      }
    }
  }
})
```

---

### 2️⃣ 请求写法

```ts
fetch('/api/users')
```

或（axios）：

```ts
axios.get('/api/users')
```

---

## 四、工作原理

```
浏览器 → Vite dev server → 后端服务器
```

- 浏览器请求的是 Vite（同源）
- Vite 再转发到后端（跨域发生在服务器之间，不受限制）

---

## 五、关键点

- 前端必须使用 `/api` 前缀
- 不要直接写后端完整地址（如 http://localhost:3000）
- proxy 只在开发环境生效

---

## 六、常见问题

### ❌ 请求没走代理

**原因：**

```
没有使用 /api 前缀
```

---

### ❌ 路径错误

```ts
rewrite: (path) => path.replace('/api', '') // ❌
```

**正确：**

```ts
rewrite: (path) => path.replace(/^\/api/, '') // ✅
```

---

### ❌ 后端接口路径不一致

**检查：**

```
/api/users → 是否被 rewrite 成 /users
```

---

## 七、一句话总结

```
通过 Vite proxy，让浏览器请求同源服务器，从而绕过跨域限制
```
