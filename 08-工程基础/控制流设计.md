## 控制流设计（Control Flow Design）

### 一、核心定义

控制流设计的本质是：

> **通过约束代码的执行路径，让逻辑更简单、可读、可预测**

---

## 二、三层结构（重点）

```text
目标（Target）
↑
原则（Principles）
↑
手段（Techniques）
```

---

## 三、目标层（Target）

### ✅ Happy Path First（快乐路径优先）

**定义：**
主流程必须是**最直、最清晰的一条执行路径**

**表现：**

* 代码从上往下可以顺读
* 核心逻辑不被 if/else 包裹
* 一眼能看出“这个函数在干嘛”

```go
// 理想结构（Happy Path）
validate
process
return result
```

---

## 四、原则层（Principles）

这些是“约束你写代码的规则”

### 1️⃣ Fail Fast（早失败）

**定义：**
一旦出现错误或非法状态，立即中断执行

**作用：**

* 防止错误污染主流程
* 减少状态复杂度

---

### 2️⃣ Flatten（扁平化）

**定义：**
控制结构必须尽量减少嵌套层级

**作用：**

* 提高可读性
* 降低认知负担

---

### 3️⃣ Happy Path First（主流程优先）

**定义：**
最核心逻辑必须处于“最外层结构”

**作用：**

* 强化代码表达能力
* 主次分明

---

## 五、手段层（Techniques）

### ✅ Guard Clause（卫语句）

**定义：**
在函数前部对异常情况进行判断，并立即 return

---

### 标准写法：

```go
if err != nil {
    return err
}
```

---

## 六、三层之间的关系（核心理解）

### 1️⃣ 手段 → 原则

Guard Clause 实现：

* ✅ Fail Fast（提前 return）
* ✅ Flatten（减少嵌套）
* ✅ Happy Path First（清理干扰）

---

### 2️⃣ 原则 → 目标

当你满足：

* Fail Fast
* Flatten
* Happy Path First

👉 自然得到：

> ✅ 清晰的 Happy Path（目标达成）

---

## 七、完整执行模型（建议直接记）

```text
目标：
- 让主流程（Happy Path）清晰可见

原则：
- Fail Fast（错误立即返回）
- Flatten（减少嵌套）
- Happy Path First（主逻辑优先）

手段：
- 使用 Guard Clause 提前 return，清理异常路径

结果：
- 代码呈现线性结构（Happy Path）
```

---

## 八、对比示例（非常关键）

### ❌ 反例（无控制流设计）

```go
if cond {
    if err == nil {
        process()
    } else {
        handleErr()
    }
}
```

问题：

* 嵌套严重
* 主流程不清晰

---

### ✅ 正例（原则 + 手段）

```go
if !cond {
    return
}

if err != nil {
    return err
}

process()
```

结果：

* 扁平结构
* 错误提前退出
* 主流程清晰

---

## 九、使用时的自检标准（落地）

写完代码后检查：

1. ❓ 是否有不必要的嵌套（>2层基本有问题）
2. ❓ 错误是否第一时间 return（Fail Fast）
3. ❓ 主流程是否可以“一眼读通”（Happy Path）

满足这三点：

> 控制流设计基本合格

---

## 十、ai prompt 以及如何阅读这种控制流写的代码
prompt
```
用 Go 写代码，遵循以下规则：
- 使用 Guard Clause 处理所有异常情况（参数错误、nil、error）
- 所有错误必须立即 return（Fail Fast）
- 避免嵌套（最多2层）
- 主流程（Happy Path）必须从上到下线性可读
- 不要用 else 包裹主逻辑

```


怎么阅读：
山来直接看 happy path是否正确，然后看每个guard 的处理是否是立即return的
主要就是看是否满足这三个原则：  也就是按照卫语句模式，阅读代码
* Fail Fast（提前 return）
* Flatten（减少嵌套）
* Happy Path First（清理干扰）
