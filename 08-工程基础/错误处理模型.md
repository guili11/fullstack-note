# 一、先给你一个整体认知

> **错误处理模型 = 错误的“表达方式 + 传播方式 + 分层策略”**

核心解决三件事：

```text
1. 错误怎么定义（类型）
2. 错误怎么往上抛（wrapping）
3. 不同层该看到什么错误（分层）
```

---

# 二、三层结构（建议你这样记）

```text
错误处理模型

├── 表达（Error Representation）
├── 传播（Error Wrapping）
├── 分层（Error Layering）
```

---

# 三、第一层：错误表达（Error Representation）

## 1️⃣ 两类错误必须区分

### ✅ 系统错误（Technical Error）

* 数据库连接失败
* IO错误
* 网络错误

👉 特点：

* 不可控
* 不需要直接暴露给用户

---

### ✅ 业务错误（Business Error）

* 用户不存在
* 密码错误
* 权限不足

👉 特点：

* 可预期
* 需要返回给前端

---

## 2️⃣ 基础定义方式（Go）

```go
var ErrUserNotFound = errors.New("user not found")
```

或者结构化一点：

```go
type BizError struct {
    Code    string
    Message string
}
```

---

# 四、第二层：错误传播（Error Wrapping）

这里是重点，很多人完全不会。

## 1️⃣ 什么是 wrapping

👉 **在返回错误时，附加上下文信息**

使用 Go 原生：

```go
return fmt.Errorf("get user failed: %w", err)
```

---

## 2️⃣ 为什么必须 wrapping

不 wrapping 的问题：

```go
return err
```

👉 上层只知道：错了
👉 不知道：在哪错的

---

## 3️⃣ wrapping 后

```go
return fmt.Errorf("service.GetUser: %w", err)
```

👉 错误链变成：

```text
controller → service → repo → db error
```

---

## 4️⃣ 如何判断错误类型

使用：

```go
errors.Is(err, ErrUserNotFound)
```

👉 判断“语义错误”
而不是字符串比较 ❌

---

# 五、第三层：错误分层（Error Layering）

这是你现在最该掌握的。

结合你在用的 Gin：

---

## 1️⃣ Repository 层（数据层）

```go
return fmt.Errorf("query user: %w", err)
```

👉 特点：

* 只返回技术错误
* 不关心业务语义

---

## 2️⃣ Service 层（业务层）

```go
if user == nil {
    return ErrUserNotFound
}
```

👉 特点：

* 定义业务错误
* 决定“这个错误是什么意思”

---

## 3️⃣ Controller / Handler 层

```go
if errors.Is(err, ErrUserNotFound) {
    c.JSON(404, ...)
    return
}
```

👉 特点：

* 把错误 → HTTP响应
* 不做业务判断，只做“映射”

---

# 六、完整流转示例（强烈建议你看懂）

```go
// repo
func GetUser(id string) (*User, error) {
    user, err := dbQuery(id)
    if err != nil {
        return nil, fmt.Errorf("repo get user: %w", err)
    }
    return user, nil
}

// service
func GetUser(id string) (*User, error) {
    user, err := repo.GetUser(id)
    if err != nil {
        return nil, err
    }
    if user == nil {
        return nil, ErrUserNotFound
    }
    return user, nil
}

// handler
func GetUserHandler(c *gin.Context) {
    user, err := service.GetUser(id)

    if err != nil {
        if errors.Is(err, ErrUserNotFound) {
            c.JSON(404, ...)
            return
        }
        c.JSON(500, ...)
        return
    }

    c.JSON(200, user)
}
```

---

# 七、你可以提炼的“工程规则”

```text
错误处理模型规则：

1. repo 层：
   - 只处理技术错误
   - 必须做 error wrapping

2. service 层：
   - 定义业务错误（ErrXXX）
   - 不暴露底层细节

3. handler 层：
   - 使用 errors.Is 判断错误类型
   - 映射为 HTTP 状态码

4. 全局：
   - 禁止字符串比较 error
   - 所有 error 必须可追踪（wrapping）
```

👉 错误分层崩掉

---

👉 debug 地狱
