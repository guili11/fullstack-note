# 路由参数

> GET 请求中传递参数的方式，弥补 GET 无法传递 body 的遗憾

---

## 1. `:name` 路径参数

**网址示例：** `/user/张三`

**作用：** 必须传，是路径的一部分（查某个用户）

```go
r.GET("/student/:id", func(c *gin.Context) {
    id := c.Param("id")
    c.JSON(200, gin.H{
        "id": id,
    })
})
```

---

## 2. `*path` 通配符

**网址示例：** `/file/a/b/c.png`

**作用：** 匹配多级路径，只用来做文件服务

```go
r.GET("/student/*path", func(c *gin.Context) {
    path := c.Param("path")
    c.JSON(200, gin.H{
        "path": path,
    })
})
```

---

## 3. `?` 查询参数

**网址示例：** `/student?page=1&size=10`

**作用：** 可选参数，筛选、分页、搜索专用（最常用）

```go
r.GET("/student", func(c *gin.Context) {
    page := c.Query("page") || "1"
    size := c.Query("size") || "10"
    c.JSON(200, gin.H{
        "page":   page,
        "size": size,
    })
})
```

---

## 总结

| 参数形式 | 特点 | 典型场景 |
|----------|------|----------|
| `:name` | 必传，路径的一部分 | 获取单个资源 |
| `*path` | 匹配多级路径 | 文件服务 |
| `?query` | 可选参数 | 筛选、分页、搜索 |

- query在后端路径中是没有写的，只能在前端路径中写，因为是可选参数，参数本身是有默认值的，前端可以不写。所以没有体现在后端路由中，而前两者不同，二者必须传递过来，后端才能处理，是没有默认值的。

# 路由分组，
分组概念：将路由进行分组，方便统一管理（路由前缀，加中间件，版本控制）
1. 统一前缀（版本控制 / 模块划分）
不用给每个接口重复写路径前缀，一次定义，全组生效
版本控制：/api/v1、/api/v2（前端 / 后端区分接口版本）
模块划分：/user（用户模块）、/order（订单模块）、/admin（管理员模块）
```go
// 很方便的模组划分呀， 免得繁琐的写全部路径
user:=r.Group("/user")
{
    user.GET("/photo",func(c *gin.Context))
    user.GET("/info",func(c *gin.Context))
}

cart:=r.Group("/cart")
{
    cart.POST("/list",func(c *gin.Context))
    cart.GET("/list",func(c *gin.Context))
}

// 版本控制，简单的通过路径前缀来区分版本
v1:=r.Group("/api/v1")
{
    v1.GET("/student",func(c *gin.Context))
}

v2:=r.Group("/api/v2")
{
    v2.GET("/student",func(c *gin.Context))
}
```


2. 统一挂载共性中间件（最常用！）
这是你之前学的 JWT 鉴权、日志、跨域 的最佳实践：
一组接口需要登录验证 → 给分组加 JWT 中间件
一组接口需要管理员权限 → 给分组加权限中间件
全组接口共享同一个中间件，不用每个接口单独加
```go
// 最简单的例子，JWT中间件的划分

// 受保护的接口组：必须登录才能访问
api := r.Group("/api")
{
api.Use(authMiddleware.MiddlewareFunc()) // 🔥 全组统一加JWT鉴权
api.GET("/info", getUserInfo)  // 自动鉴权
api.GET("/order", getOrder)    // 自动鉴权
}

// 公开接口组：不用登录
auth := r.Group("/auth")
{
auth.GET("/login", login)    // 无鉴权
}
```

- question:我看这个分组，是必须有一个 路由前缀的，有不带路由前缀的吗?
但是肯定得带啊，路由管理器内部就是依靠路径区分的呢，实际开发具体是如何的呢
实际开发不是 就非黑即白，按实际情况来划分的，
一般api前缀是私有的，需要统一加JWT，
auth是 登录注册这些，分组，不需要JWT

注意：不要用组别对应中间件，组别只是为了划分接口，不同组别可以use同一个中间件，但是因为有不同业务区分，所以要分组。（否则，路径就一连串路径了，/private/user/login /private/order/list）
