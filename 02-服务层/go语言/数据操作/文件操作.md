# 文件操作

在开始学习文件操作之前，先想好一个问题：

## 文件特征

文件本身有什么特征会影响我们的文件操作？

1. **文件的大小** —— bufio，流式读取
2. **文件的格式** —— json、yaml、xml等不同格式的文件需要不同的 decoder
3. **文件的结构** —— 需要自定义好结构体

**次重要：**

4. 文件的编码格式（UTF-8、GBK、Big5等）
5. 文件的校验

## 快速入门

```go
// 读：xxx.NewDecoder(file).Decode(&data)  ,xxx.NewWriter(file).Read(&data)
// 写：xxx.NewEncoder(file).Encode(data)   ,xxx.NewWriter(file).Write(data)  flush()
```

## 基础概念

开始文件操作，先说好：

- **从物理位置上看**：文件操作就是将数据从磁盘读取到内存，或者反过来
- **从逻辑上看**：文件操作是将数据从一定的文件格式读取为一定的数据结构（字符串，对象等），或者反过来

那么具体如何读取呢？

- 实则就三步：打开文件 → 读取文件（解码文件内容）→ 关闭文件
  1. 打开文件，获取到 File 对象
  2. 读取文件内容，提供数据结构以及容器 data
  3. 关闭文件 defer，注意在打开成功时才有 defer

**最关键的两个数据**：File 对象以及最终的 data 结构容器

## 文件操作函数

### 1. 打开文件

- `os.OpenFile`：打开文件，获取到 File 对象

```go
func OpenFile(name string, flag int, perm os.FileMode) (*os.File, error)
func Open(name string) (*os.File, error)  // 简单打开文件方法
```

### 2. 读取文件

- `Read`：读取容器大小的文件内容，调用者必须提供一个容器，用于存储读取到的内容（指针）
- `ReadAll`：读取整个文件内容，直接返回一个字节切片

```go
func (f *os.File).Read(b []byte) (n int, err error)
func ReadAll(r io.Reader) ([]byte, error)
```

> 注意：这里我们需要提供 File 对象以及 data 结构容器，这里的调用不是传递两个参数，而是使用 File 对象的 Read 方法，这里不是包函数哦，而是对象方法

### 3. 解码器

- `xxx.NewDecoder`：xxx 格式类型的解码器

```go
func xxx.NewDecoder(r io.Reader) *Decoder
```

> 注意：这里参数类型是接口 `io.Reader`，File 对象实现了这个接口，所以可以直接传递 File 对象。接口的写法使得，这里的解码器可以处理任何想要处理的数据源，例如网络请求体: `http.Request.Body`

#### 理解：原始解码流程

实际上呢，解码应该是读取文件后才进行的，原本的解码，例如 json 格式解码是这个样子的：

```go
// 1. 打开文件
file, _ := os.Open("config.json")
defer file.Close()

// 2. 先读取：将文件内容全部读入内存的 byte 切片中
content, _ := io.ReadAll(file)

// 3. 后解码：将字节切片转换为结构体
var config Config
err := json.Unmarshal(content, &config)
```

回忆在最开始我们提到的文件重要的两点：文件大小以及文件的格式。假如文件太大了，这里 `ReadAll` 就会瞬间占满内存，出问题，所以需要流式读取。然后对于不同的文件格式，需要不同的解码方法调用。

#### Decoder 的诞生

- **提 (Provide)**：提取/打开数据源（File 对象）
- **连 (Link)**：实例化解码器，将源与规则连接（`xxx.NewDecoder`）
- **转 (Transform)**：执行转换，数据直接从源流向结构体（`decoder.Decode`）

##### 不同格式只需要去调用不同的解码器就可以：

- 他们接口一致，方法一致，只是包名不同
- 不同的文件格式的时候只需要替换包名就可以了

##### 逻辑汇总

只需要关注两个对象，一个容器：File 对象、decoder 对象、数据结构容器

##### 使用方法

只需要提供 File 对象以及数据结构容器，解码器会自动读取文件内容，并且解码到数据结构中

**例如：**

```go
// 1. 打开文件 (file 此时就是一个 io.Reader)
file, _ := os.Open("config.json")
defer file.Close()

// 2. 实例化解码器：直接关联数据源
decoder := json.NewDecoder(file)
decoder := yaml.NewDecoder(file)  // 就可以直接解码 yaml 格式的文件了

// 3. 执行解码：直接从 file 中提取数据并填充容器
var config Config
err := decoder.Decode(&config)
```

### 4. 流式操作

既然提到了流式操作，这里介绍一下 go 原生的文件流式操作方法：

> 注意：虽然 decoder 具备流式的能力，但是有时我们不需要其中的解码，所以这里深入更加底层的流式操作

#### bufio 包

这里读取有 `bufio.NewReader(file)` 以及 `bufio.NewScanner(file)` 两种方式：

- 前者可以指定读取模式，例如按行读取、按字节读取等
  - `reader.ReadString('\n')`
  - `reader.ReadBytes('\n')`
  - `reader.ReadLine()`
  - 这些方法都可以实现按行读取
- 后者默认就是按行读取，提供了 `Scan` 方法来迭代读取每一行
  - `scanner.Scan()`

```js
 const reader = response.body.getReader();

    // 创建一个文本解码器来处理流数据 
    const decoder = new TextDecoder();
    let fullText = "";

    while (true) {
      // 1.读取流数据块
      const { done, value } = await reader.read();
      // done标志着流的结束，value是当前块数据
      if (done) break;
      // 2.解码当前块数据（字节数据->字符数据）
      const chunk = decoder.decode(value);
    }
```

```go

file,_:=os.Open("test.txt")
defer file.Close()

// 底层流式读取，允许在流式读取时，执行一些操作，读取的格式一般都是简单格式，例如txt
bufScanner:=bufio.NewScanner(file)
for bufScanner.Scan(){
  line:=bufScanner.Text()
  fullText+=line
}

// 这种包含解码的流式读取，数据全部解析完了才能用， 所以没有什么流式读取时操作一说，只能等数据全部解析完了，才能使用
var config Config
err := decoder.Decode(&config)



```

> 注意一点：读操作不需要管缓冲区，因为我想读就可以从缓冲区中读到，就算还有剩余，那也是因为我自己不读了；但是写操作，是缓冲区满了才会写入文件，假如我最后没写满，那这些数据就不会写入文件了，所以写操作需要调用 `Flush` 方法来强制将缓冲区中的数据写入文件

#### 两种模式对比

发现获取到 File 对象后，有两种模式：

- **直接开干**：`io.ReadAll(file)` && `file.Read(&data)`
- **构造器+方法**：`reader := bufio.NewReader(file)` && `reader.ReadString('\n')` && `decoder := json.NewDecoder(file)` && `decoder.Decode(&data)`

道理很好理解呀，就是因为我实现了一些额外的功能，例如缓冲等（我需要在构造函数中初始化一些状态）。这里跟 axios 那里也是一致的，可以这么说，具备额外功能的地方，就是需要 **构造器+方法** 的设计模式

### 5. 写入文件

写入嘛，很好理解，就是写东西呗。

- `Write`：写入文件内容，原生Write方法需要自己按照文件格式决定写入的内容的格式，例如csv，  包方法，就简单一些，只需要提供要求格式的内容即可，而且有性能优化（一般就是加了一个缓冲区）

```go
func (f *os.File).Write(b []byte) (n int, err error)
func (f *os.File).WriteString(s string) (n int, err error)  // 方便直接写字符串
```

**示例：**

```go
file, _ := os.OpenFile("test.txt", os.O_WRONLY|os.O_CREATE, 0666)
defer file.Close()

file.Write([]byte("hello world"))
```

#### 带缓冲的写入

```go
// 1. 构造层：绑定文件对象，开辟缓冲区
writer := bufio.NewWriter(file)

// 2. 方法层：写入到内存缓冲区
writer.WriteString("hello")
writer.Write([]byte("world"))

// 3. 必须调用的核心方法：清空/冲刷
writer.Flush()  // 将内存中还没满的数据强制推送到磁盘
```

**无敌：一遍输出一边写入**

```go
fmt.Fprintf(file, "名字：%s，年龄：%d\n", "Alice", 20)
```

### 6. 模式一致性

- **读**：`File -> Decoder -> Struct`
- **写**：`Struct -> Encoder -> File`

可以看出来了，实际上，还有编码器的存在，encoder。

**示例：**

```go
// 1. 提 (Provide): 准备好目的地 (实现了 io.Writer 的 file)
file, _ := os.Create("data.json")
defer file.Close()

// 2. 连 (Link): 实例化编码器，绑定目的地
encoder := json.NewEncoder(file)

// 3. 转 (Transform): 直接将结构体压成 JSON 流推送到文件
data := Config{Name: "Copilot", Version: 1.0}
encoder.Encode(data)
```
