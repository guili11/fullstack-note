# 网络操作

## 分类

网络操作分两种（默认都是 HTTP 请求）：

- **Client 操作**
- **Server 操作**

由于 HTTP 的特性：一发一返，Client 操作和 Server 操作是对应的。

## 纲领

聚焦：**数据 + 网络行为控制**（业务 + 网络）

- 准备头
- 体（其中也包含网络行为控制）
- 控制网络行为（超时等）

> 「请求头控制」是「业务 / 协议规则的定义」\
> 「超时控制」是「网络连接 / IO 生命周期的约束」

## 职责划分

以 axios 为例：

- `createAxios` 配置：网络连接行为（超时啊，跨域配置啊等）
- `axios.get(url, data)`：业务请求行为

## Client 操作 vs Server 操作

| 操作类型   | 行为                            |
| ------ | ----------------------------- |
| Client | 准备请求头、请求体、发送 req              |
| Server | 接收 req、准备 res（响应头、响应体）、发送 res |

### 响应差异

- **Client 响应**：主要体现在 HTML 上
- **Server 响应**：主要体现在对底层数据库的操作，以及重新发起的 HTTP 请求

## Server 端行为分析

### 核心流程

接收 req → 准备 res → 发送 res

### 核心关注点

- **核心**：读取 req、准备 res 数据
- **补充**：配置网络行为（响应头 + 超时等网络配置）

> 注：网络行为配置属于全局配置，在创建 axios 实例时配置。

## 读 + 写操作

### 核心发现

读 + 写这两个操作，其实跟文件操作是一样的。

**数据操作是一家**，只不过 Server 端是通过 HTTP 请求获取到 body，这个其实就是 file。

二者都是数据流，都实现了 `io.Reader` 和 `io.Writer` 接口，因此可以传入统一的体系操作：

```go
// 读
xxx.NewDecoder(file/body).Decode(&data)
xxx.NewWriter(file/body).Read(&data)

// 写
xxx.NewEncoder(file/body).Encode(data)
xxx.NewWriter(file/body).Write(data)
flush()
```

## 通用操作对比

| 通用操作行为  | 本地文件操作 (`os.File`)        | HTTP Body 操作 (`Request/Response`) |
| ------- | ------------------------- | --------------------------------- |
| 打开流     | `os.Open` / `os.OpenFile` | `req.Body` / `resp.Body`          |
| 读取数据    | `file.Read(b []byte)`     | `body.Read(b []byte)`             |
| 创建格式解码器 | `json.NewDecoder(file)`   | `json.NewDecoder(req.Body)`       |
| 解码到数据结构 | `decoder.Decode(&v)`      | `decoder.Decode(&v)`              |
| 写入数据    | `file.Write(b []byte)`    | `resp.Write(b []byte)`            |
| 关闭流     | `file.Close()`            | `body.Close()`                    |
