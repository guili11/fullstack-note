RESTful = 用一套统一、规范、约定好的规则，设计前后端接口。

## 规则核心

用 URL 定位资源，用 HTTP 方法表示动作，用状态码表示结果。

---

## 详细说明

### 1. 资源（Resource）

就是你要操作的 "东西"：用户、文章、订单、商品、视频……

用 URL 表示它，比如：
- `/users`
- `/articles`
- `/orders`

### 2. 动作（HTTP 动词）

不用在 URL 里写 `getUser`、`deleteOrder`以代表动作行为，直接用 HTTP 方法表示你要干嘛：

| 方法 | 含义 |
|------|------|
| GET | 查 |
| POST | 增 |
| PUT | 全量改 |
| PATCH | 部分改 |
| DELETE | 删 |

### 3. 结果（HTTP 状态码）

不用永远返回 200，直接用状态码告诉前端成没成：

| 状态码 | 含义 |
|--------|------|
| 200 | 成功 |
| 201 | 创建成功 |
| 401 | 没登录 |
| 403 | 没权限 |
| 404 | 找不到 |
| 500 | 服务器炸了 |

## 进阶：接口版本控制

接口改了，老前端不能崩，所以要加版本：

**最简单：**
- `/v1/users`
- `/v2/users`

**最优雅：**
放在请求头里，不污染 URL
