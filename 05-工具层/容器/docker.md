

docker的作用：
我们的代码可不是一个exe，什么环境都不需要，直接就跑。不同语言的代码需要不同的环境，例如js需要node或者浏览器环境
go需要go环境...   java这种虚拟机方便的，实则也是需要java环境的，有时候同一个go，不同的版本，需要不同的环境。这就很难办，
之前，我们用虚拟机解决这个问题，现在进化到了docker。

镜像，容器介绍：
镜像就是准备好的应用程序，容器就是运行的应用程序实例。
运行的容器就是一个隔离的进程，拥有自己的文件系统，网络。
我们外界跟容器交流主要通过三个：
一个是文件挂载，一个是环境变量，一个是端口映射。
文件挂载：持久化容器数据
环境变量：配置容器运行环境
端口映射：将容器的端口映射到宿主机的端口，方便外部访问。

如何制作镜像，以及开启容器呢？
镜像制作：Dockerfile  + docker build
容器开启：docker run -d  image-name  （运行一个镜像就是开启一个独立的容器）

Dockerfile:镜像组装的说明书，那么组装镜像，我们需要考虑哪些东西呢?
实际上就是考虑我的代码运行需要什么环境，以及需要哪些依赖。
环境+依赖+

FROM WORKDIR COPY  RUN CMD  核心就这五个。

1. 基础镜像：选择一个基础的镜像，作为我们的基础。
2. 复制代码：将我们的代码复制到镜像中。
3. 安装依赖：安装我们的依赖，例如node、python等。
4. 配置环境变量：配置我们的环境变量，例如PORT等。
5. 开启端口：开启我们的端口，例如8080等。
6. 运行程序：运行我们的程序，例如npm start等。

概念	说明
镜像（Image）	只读模板，由多层文件系统叠加而成；包含应用、运行时、库与配置等。
容器（Container）	镜像的运行实例，在镜像之上有一层可写容器层（就是volume）；删除容器会丢掉未持久化到卷中的可写数据（volume以外的）。
Dockerfile	描述「如何从基础镜像一步步构建出目标镜像」的文本文件。
仓库（Registry）	存放镜像的服务；默认使用 Docker Hub 等。镜像名常写作 仓库/镜像:标签，如 mysql:8.0。
Docker Engine	宿主机上的守护进程，负责拉取镜像、创建与管理容器。
Docker CLI	命令行客户端（docker），通过 API 与 Engine 通信。

什么是容器？简而言之，容器是应用程序每个组件的隔离进程。每个组件——前端 React 应用程序、Python API 引擎和数据库——都在其自身的隔离环境中运行，与您机器上的其他所有内容完全隔离。
特点：自包含，**隔离**，独立，可移植
容器与虚拟机， 虚拟机是一个隔离的操作系统，容器是一个隔离的进程（前者开销大，后者开销小）

那么进程对应的应用程序是什么？   其实就是镜像了哦
镜像：只读模板，由多层文件系统叠加而成
只读：模板的设计就是只读的，类似于一个库，你拿来直接用，你不可以修改它的内容，但是可以扩展它
叠加：实际是将基础镜像复用，这样可以省空间，例如我有两个镜像，他们俩都用到了一个基础的nginx镜像，那么我分层之后，可以只拉取一次，然后复用这个基础镜像。

Dockerfile你手写的组装说明书。
规定好：用什么环境、复制哪些代码、装依赖、开什么端口、最后运行你的程序。
你的代码 / 项目文件
前端、Go、Java、Python 代码、静态页面、配置文件，都是你自己的业务。
基础镜像（缺一不可）
Dockerfile 第一行 FROM xxx比如 FROM nginx、FROM golang:1.23、FROM node:20它自带：迷你系统、运行时、底层依赖，不用你从零装系统。

从-v这里就可以看出来，容器名字的由来。


容器端口映射：
由于容器是隔离的，所以容器里监听的端口，是容器内部的端口，不是宿主机的端口。
所以，我们要在宿主机上访问容器里的服务，需要映射端口。

外部请求 → 宿主机 IP:宿主机端口 → 转发 → 容器 IP:容器端口

映射端口：docker run -p 宿主机端口:容器内端口

1. 端口存在的核心作用
   一台电脑只有 1 个 IP，但是同时运行一堆程序：微信、浏览器、游戏、你的 Go 服务、数据库、Docker 容器……
   👉 如果没有端口：所有网络数据全部乱套，电脑不知道数据该发给哪个软件。

端口 = 给不同程序划分专属「接收门」，分流网络数据

监听8080的意思：任何请求到8080的，交给我处理

程序正常运行，根本不需要端口
只有「要接收别人请求」的服务，才需要 监听端口

镜像运行配置项

`-d`：后台跑

&#x20;

- `-p`：端口映射（外部访问）
- `--name`：起名字（通信 + 管理）
- `-v`：挂文件（改配置不用重做镜像）
- `--network`：容器互通（内网连接）
