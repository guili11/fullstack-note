# Redis 核心极简关键点
## 1. Redis 是什么
- 一款**内存数据库**，key-value 键值对存储
- 速度极快，**主要用来做缓存**，辅助数据库
- 非关系型（NoSQL），不像 MySQL 存表、行、列

## 2. 为什么这么快
1. 数据存在**内存**，不是硬盘
2. 核心工作**单线程**，没有锁竞争开销
3. 高效 IO 模型 + 简单数据结构

## 3. 5 种最常用数据类型（记住名字就行）
1. **String**：字符串（普通缓存、验证码、token）
2. **Hash**：哈希（存用户/商品对象）
3. **List**：列表（消息队列、有序列表）
4. **Set**：集合（去重、共同好友）
5. **ZSet**：有序集合（排行榜）

## 4. 项目里用来干什么（必考）
- 热点数据**缓存**（减轻 MySQL 压力）
- 短信验证码、登录令牌
- 点赞、浏览量、计数器
- 分布式锁、接口限流
- 临时数据、限时数据

## 5. 两个关键机制
1. **过期时间**
key 可以设过期，自动销毁，适合验证码、临时缓存
2. **持久化**
本来是内存断电就丢，Redis 可以存到硬盘，断电不丢数据

## 6. 和 MySQL 区别（秒懂）
- MySQL：存永久数据、存硬盘、慢、复杂查询
- Redis：存临时热点、内存读写、超快、只做简单读写
